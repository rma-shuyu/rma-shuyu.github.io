什么是iptables?

iptables是一个在Linux系统中使用的防火墙工具。它可以使用规则来过滤、屏蔽、重定向和修改数据包。iptables在网络安全方面扮演了很重要的角色。
详情参考

基本命令

iptables -L - 查看当前的iptables规则
iptables -A - 添加一条规则
iptables -D - 删除一条规则
iptables -P - 设置规则的默认策略
iptables -F - 清除所有规则

iptables的规则结构

表（Table）：标识要执行哪个表的规则。默认情况下，iptables使用filter表。
链 （Chain）：包含一组规则的有序列表。iptables中预定义的三个链分别是INPUT、OUTPUT和FORWARD。
匹配条件（Match）：iptables会将收到的每个数据包使用匹配条件来测试是否与规则匹配。例如，匹配源IP地址、目的IP地址、端口等。
动作（Target）：匹配成功后要执行的操作。例如，接受、拒绝、重定向等。

命令选项输入顺序

iptables表

filter
这是iptables中默认的表。它用于过滤数据包。当数据包通过iptables时，它会被传递到filter表的三个默认链中的一个：INPUT、OUTPUT或FORWARD。

nat
这个表被用于网络地址转换。它可以用来隐藏内部网络的IP地址，或将IP地址转换为其他地址。如果你需要在服务器上配置NAT网关，这个表会很有用。

mangle
这个表可以修改数据包头，如修改TTL值、标记数据包等。

raw
这个表涉及到一些底层的网络细节，它用来禁用NAT记号和连接追踪功能。你不应该使用这个表除非你非常清楚自己在做什么。

security
这个表是CentOS和Red Hat Enterprise Linux的SELinux规则，通常不需要手动设置或修改它。

iptables链

INPUT
这个链处理服务器收到的数据包。例如，当你远程连接到服务器时，数据包通过INPUT链进入iptables。

OUTPUT
这个链处理从服务器发出的数据包。例如，当你从服务器下载文件时，数据包通过OUTPUT链离开iptables。

FORWARD
FORWARD链用于路由数据包。当一个数据包应该从一个接口转发到另一个接口时，这个链就会被用到。

PREROUTING
在路由之前更改数据包。即在数据包到达系统之后（并且在路由之前）立即发生数据包转换。这有助于将数据包的目标 IP 地址转换为与本地服务器上的路由匹配的内容。这用于 DNAT（目标 NAT）。

POSTROUTING
在路由后更改数据包。即当数据包离开系统时发生数据包转换。这有助于将数据包的源 IP 地址转换为可能与目标服务器上的路由匹配的内容。这用于 SNAT（源 NAT）。

iptables匹配条件

-p, –protocol
指定使用哪个协议。例如，TCP、UDP、ICMP等等。

-s, –source
指定源IP地址或地址段。可以使用CIDR的格式指定IP地址段。

-d, –destination
指定目标IP地址或地址段。同样也可以使用CIDR格式。

-i, –in-interface
指定进入数据包的接口。

-o, –out-interface
指定离开数据包的接口。

-m, –match
指定特定的匹配条件模块。这个参数允许你使用额外的匹配条件，例如mac地址、时间、字符串和其他更高级的条件。

iptables动作

ACCEPT
接收数据包

REJECT
拒绝数据包，发送一个ICMP消息通知。

DROP
静默地丢弃数据包，不发送任何消息。

LOG
记录数据包到日志中并继续处理规则。

RETURN
从一个自定义链返回。
